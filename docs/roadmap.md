# FirmClaw — 类 OpenClaw 智能体项目开发计划

## 项目概述

FirmClaw 是一个从零搭建的本地优先 AI 智能体框架，核心采用 ReAct（Reasoning + Acting）架构。
灵感来源于 OpenClaw，目标是实现一个可理解、可扩展、本地运行的智能体系统。

## 核心本质

OpenClaw 的核心可以用一句话概括：**一个基于 ReAct 循环的 LLM 工具调用引擎**。

灵魂组件按优先级排列：

1. **Agent Loop（ReAct 循环）** — 绝对核心，LLM 思考 → 调工具 → 观察结果 → 继续思考
2. **工具系统** — 手和脚，极简设计（read/write/edit/bash）
3. **会话管理** — 记忆的骨架，树状 JSONL 结构
4. **系统提示词** — 大脑的编程方式，动态组装
5. **上下文压缩** — 生存机制，三级递进
6. **记忆系统** — 进阶能力，混合检索
7. **网关层** — 扩展能力，多平台适配

## 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | TypeScript + Node.js | 生态丰富，OpenClaw 同款 |
| LLM 提供商 | Kimi (Moonshot AI) | 兼容 OpenAI API 格式 |
| 包管理 | pnpm | 快速、节省磁盘 |
| 存储 | JSONL 文件 + SQLite | 本地优先，简单可靠 |
| 向量搜索 | sqlite-vec（后续阶段） | 本地运行，无需外部服务 |
| 实时通信 | WebSocket（后续阶段） | 双向流式通信 |

## 实施路线（6 个阶段）

### Phase 1: 最小可用的 ReAct 循环

**目标**: 能跑通一个最简单的 Thought → Action → Observation 循环

**实现内容**:
- [ ] 项目初始化（pnpm + TypeScript）
- [ ] LLM 调用封装（Kimi API，兼容 OpenAI 格式）
- [ ] Agent Loop: while 循环 + 工具调用解析
- [ ] 事件流机制（EventEmitter，流式输出）
- [ ] 第一个工具：`bash`（命令执行）
- [ ] CLI 入口（命令行交互）

**验证标准**: 用户输入"列出当前目录文件"，智能体能调用 bash 执行 `ls` 并返回结果

---

### Phase 2: 完善工具系统

**目标**: 4 个核心工具完整实现

**实现内容**:
- [ ] `read` 工具（文件读取，支持 offset/limit）
- [ ] `write` 工具（文件写入，支持大文件分块）
- [ ] `edit` 工具（查找替换，唯一性校验）
- [ ] `bash` 工具完善（超时、流式输出、工作目录）
- [ ] 工具注册机制（统一接口 + JSON Schema 参数校验）
- [ ] 基础权限策略（allowlist/denylist）

**验证标准**: 智能体能读取代码文件、编辑文件、执行构建命令

---

### Phase 3: 会话管理与系统提示词

**目标**: 支持多轮对话

**实现内容**:
- [ ] 会话存储（JSONL，append-only）
- [ ] 会话上下文重建（从 JSONL 回溯构建 messages 数组）
- [ ] 系统提示词组装（动态注入工具定义、运行时信息）
- [ ] 工作区文件加载（SOUL.md / AGENTS.md / MEMORY.md）
- [ ] 基础上下文窗口管理（token 计数 + 简单截断）

**验证标准**: 多轮对话中，智能体能记住之前的上下文并持续工作

---

### Phase 4: 上下文压缩与记忆系统

**目标**: 长对话不丢失关键信息

**实现内容**:
- [ ] 上下文裁剪（旧 tool result 裁剪）
- [ ] LLM 摘要压缩（调用 LLM 生成历史摘要）
- [ ] 记忆文件管理（MEMORY.md 的读写）
- [ ] BM25 全文搜索（基于 SQLite FTS5）
- [ ] 向量搜索（sqlite-vec，可选）

**验证标准**: 长对话（50+ 轮）中，智能体不会遗忘关键决策

---

### Phase 5: 安全与进阶特性 ✅

**目标**: 可靠的生产级系统

**实现内容**:
- [x] 人工审批流程（Human-in-the-Loop）— `v4.1.0`
- [x] Prompt Injection 防护（扫描工具结果）— `v4.2.0`
- [x] 审计日志 — `v4.3.0`
- [x] Heartbeat 自主循环 — `v4.4.0`
- [x] 会话分支 — `v4.5.0`
- [x] 工具执行钩子（before/after hook）— `v4.5.0`

**验证标准**: 智能体能自主运行定时任务，危险操作需要人工确认 — **通过**

---

### Phase 6: 网关与多平台适配

**目标**: 对外提供服务

**实现内容**:
- [ ] WebSocket 服务器 — `v5.1.0`
- [ ] 消息路由与会话解析 — `v5.1.0`
- [ ] CLI 交互界面优化 — `v5.2.0`
- [ ] 子智能体（Subagent）能力 — `v5.3.0`
- [ ] Web UI（可选） — `v5.4.0`
- [ ] 全量整合发布 — `v6.0.0`

**详细设计**: 参见 [`roadmap-phase6.md`](./roadmap-phase6.md)

**验证标准**: 通过 CLI 或 Web 界面与智能体交互
