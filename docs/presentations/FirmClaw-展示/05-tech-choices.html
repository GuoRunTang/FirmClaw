技术选型决策说明
逐一解释 FirmClaw 的每个技术决策——为什么这样选，有什么替代方案

设计原则：每个技术选型都遵循三个标准：简单（新人容易理解）、可靠（生产级稳定性）、自主（不依赖外部服务）。

一、编程语言：TypeScript + Node.js

对比维度TypeScript + Node.jsPythonGo
类型系统✅ 静态类型，编译时检查⚠️ 可选（mypy），社区不强制✅ 静态类型
异步处理✅ async/await 原生支持✅ asyncio✅ goroutine
LLM SDK 生态✅ OpenAI SDK（官方 TypeScript）✅ OpenAI SDK（官方 Python）⚠️ 社区维护
参考项目✅ OpenClaw 同款语言❌ OpenClaw 不用 Python❌ 无参考项目
JSON 处理✅ 原生 JSON 对象⚠️ 需要序列化/反序列化⚠️ 需要序列化
运行时⚠️ 需要 Node.js⚠️ 需要 Python✅ 单二进制

✅ 优势：与 OpenClaw 同款语言，可以直接参考其实现；LLM SDK 生态成熟；JSON 处理天然友好；静态类型减少运行时错误。

❌ 劣势：Python 开发者需要学习新语言；Node.js 单线程模型对 CPU 密集任务不利。

决策：选择 TypeScript + Node.js，主要因为 与 OpenClaw 保持技术栈一致，便于参考和移植。如果用 Python，就无法直接参考 OpenClaw 的代码实现。

二、存储格式：JSONL 文件

对比维度JSONL 文件SQLitePostgreSQL
复杂度✅ 零配置⚠️ 需要引入 sqlite3❌ 需要外部服务
可读性✅ 文本编辑器直接查看⚠️ 需要 SQL 查询❌ 需要 SQL 查询
Git 友好✅ diff 友好（每行一条记录）⚠️ 二进制文件❌ 外部数据
并发写入⚠️ 需要文件锁✅ 内置锁机制✅ 完善的事务
查询能力❌ 只能全文件扫描✅ SQL 查询✅ SQL + 索引
追加性能✅ O(1) append✅ 快速 INSERT✅ 快速 INSERT

✅ 优势：零配置、零依赖、可直接用文本编辑器查看、Git 可追踪变更、追加写入性能极佳。

❌ 劣势：查询能力弱（只能全文件扫描）；不支持并发写入；不适合大数据量。

决策：选择 JSONL，因为 FirmClaw 是个人 Agent（单用户、单进程），不需要数据库的事务和并发能力。"能用文件就不用数据库"是本地优先架构的核心原则。

三、搜索引擎：纯 JS BM25

对比维度纯 JS BM25SQLite FTS5Elasticsearch向量搜索
外部依赖✅ 零依赖⚠️ 需要 sqlite3❌ 需要独立服务⚠️ 需要 embedding 模型
中文支持✅ 自实现 bigram✅ 内置 tokenizer✅ 插件支持✅ embedding 自带
语义理解❌ 仅关键词匹配❌ 仅关键词匹配⚠️ 有限✅ 语义相似度
索引持久化✅ JSON 文件✅ SQLite 数据库✅ 内置✅ 向量数据库
部署复杂度✅ 零部署⚠️ 需要安装❌ 需要集群❌ 需要模型+数据库
决策：选择纯 JS BM25，因为零外部依赖。对于 FirmClaw 当前的记忆规模（几十到几百条），BM25 完全够用。未来版本（v7.2）会在此基础上叠加向量搜索，形成"关键词 + 语义"的混合检索。

四、LLM 接入：OpenAI 兼容 SDK

几乎所有的 LLM 提供商（DeepSeek、Kimi、MiniMax、Claude、通义千问）都提供了 OpenAI 兼容的 API 格式。这意味着：

  • 用同一个 SDK（openai）就能调用所有模型
  • 切换模型只需要改 base_url 和 api_key
  • 本地模型（Ollama）也兼容这个格式

唯一例外：Claude 的官方 API 格式与 OpenAI 不同，但可以通过第三方代理（如 OpenRouter）转换为 OpenAI 格式。

决策：使用 openai npm 包作为 LLM 接入层。这是当前 AI 生态的事实标准，最大程度减少模型切换的成本。

五、实时通信：WebSocket（ws 库）

对比维度WebSocket (ws)SSE (Server-Sent Events)HTTP 轮询
双向通信✅ 服务端和客户端都能主动发消息❌ 仅服务端→客户端❌ 仅客户端→服务端
实时性✅ 毫秒级✅ 毫秒级❌ 取决于轮询间隔
复杂度⚠️ 需要管理连接状态✅ 简单（HTTP 之上）✅ 最简单
适用场景双向实时交互（审批请求等）单向推送（事件流）低频查询
决策：选择 WebSocket，因为 Agent 交互需要双向实时通信——不仅服务端要推送事件（思考内容、工具状态），客户端也要发送消息和审批响应。SSE 和 HTTP 轮询无法满足这个需求。

六、参数校验：ajv（JSON Schema）

LLM 传来的工具调用参数可能包含错误类型、缺失字段、非法值。ajv 是 JavaScript 生态中最流行的 JSON Schema 校验库，功能等价于 Python 中的：

  • pydantic — Python 数据校验
  • cerberus — 轻量级校验
  • jsonschema — JSON Schema 校验

选择 ajv 而非其他校验库，因为它是 Node.js 生态中性能最高、兼容性最好的选择。

决策：使用 ajv，这是 Node.js 生态的标准选择，类似 Python 中用 pydantic 做参数校验。

一句话总结：FirmClaw 的所有技术选型都遵循"能用简单方案就不用复杂方案"的原则。如果某个选型让你觉得"这也太简陋了"，请记住——Agent 的核心价值在于 LLM 的推理能力，而非基础设施的复杂性。简单的基础设施 + 强大的 LLM = 最好的 Agent。
