三、Agent Loop 实现详解

Agent Loop 是 FirmClaw 的心脏，在 src/agent/agent-loop.ts 中实现，约 200 行代码。

核心数据流

# agent-loop.ts 的逻辑（Python 伪代码） class AgentLoop: def run(self, user_input: str) -> str: # 1. 恢复历史消息 messages = session_manager.load_messages() # 2. 构建系统提示词（动态组装） system_prompt = context_builder.build( soul=read_file(".firmclaw/SOUL.md"), tools=registry.get_definitions(), memory=search_engine.search(user_input) ) # 3. ReAct 循环 messages.append({"role": "user", "content": user_input}) while True: # 3a. 上下文压缩（如果需要） messages = compress_if_needed(messages) # 3b. 调用 LLM response = await llm.chat(messages, tools, on_delta=emit_thinking) # 3c. 如果没有工具调用，返回最终回复 if not response.tool_calls: return response.text # 3d. 执行每个工具调用 for call in response.tool_calls: result = await registry.execute( call.name, call.args, context ) messages.append({"role": "tool", "content": result})

四、工具系统实现详解

工具系统在 src/tools/ 目录下，核心是 registry.ts（注册中心）和 4 个工具实现。

4.1 工具定义（类似 Python 的函数签名）

# tools/types.ts 中的工具接口（Python 等价） class Tool: name: str # 工具名，如 "bash" description: str # LLM 看到的功能描述 parameters: dict # JSON Schema 格式的参数定义 execute: Callable # 实际执行函数

4.2 工具执行流水线

registry.ts 的 execute() 方法是所有工具的统一入口，执行 7 个步骤：

1. 参数校验（ajv）

类似 Python 的 pydantic.validate_arguments，用 JSON Schema 校验 LLM 传来的参数是否合法

2. Before Hooks

执行前钩子，可修改参数或拒绝执行

3. 权限检查

路径白名单 + 命令黑名单 + 风险评估

4. 人工审批

高风险操作（如 rm）暂停等待用户确认

5. 工具执行

调用实际的 bash/read/write/edit 函数

6. After Hooks

执行后钩子：审计日志、结果处理

7. Prompt Injection 扫描

扫描返回结果中的注入攻击，标记可疑内容

4.3 bash 工具详解

# tools/bash.ts 的核心逻辑（Python 等价） async def execute_bash(command: str, cwd: str, timeout: int) -> str: # 类似 Python 的 subprocess.Popen process = spawn(command, {cwd, timeout}) # 流式收集输出（逐块读取，而非等全部完成） output = "" for chunk in process.stdout: output += chunk emit_event("bash_output", chunk) # 实时推送 # 超时处理（类似 Python 的 threading.Timer） if timeout_exceeded: process.kill() # 发送 SIGTERM return output

五、LLM Client 实现详解

在 src/llm/client.ts 中，封装了与 LLM API 的通信。核心设计是统一 OpenAI 兼容格式。

# llm/client.ts 核心逻辑（Python 等价） from openai import AsyncOpenAI class LLMClient: def __init__(self, api_key, base_url, model): # 所有模型提供商统一用 OpenAI SDK self.client = AsyncOpenAI(api_key, base_url) self.model = model # "deepseek-chat", "gpt-4", 等 async def chat(self, messages, tools=None, on_delta=None): response = await self.client.chat.completions.create( model=self.model, messages=messages, tools=tools, # 工具定义（让 LLM 知道可用什么工具） stream=True # 流式输出 ) # 逐块读取流式响应，调用 on_delta 回调 async for chunk in response: if on_delta: await on_delta(chunk) return response

关键点：MiniMax、DeepSeek、Kimi、Claude 等都提供了 OpenAI 兼容的 API。只需改 base_url 和 api_key，代码完全不变。这就像 Python 中用同一个 requests 库调用不同 REST API。

七、网关层实现详解

网关层在 src/gateway/ 目录下，基于 WebSocket + JSON-RPC 2.0 协议。

WebSocket 服务器

基于 ws 库（类似 Python 的 websockets），接收 JSON-RPC 请求，路由到对应的处理函数。

Python 等价：websockets.serve(handler, port=3000)

JSON-RPC 协议

标准的请求/响应格式，方法包括 agent.chat、session.list、approval.respond 等。

Python 等价：可以用 jsonrpcserver 库实现

事件推送

AgentLoop 的所有事件（思考内容、工具开始/结束、审批请求）通过 EventEmitter → WebSocket 实时推送给客户端。这样 Web UI 和 CLI 可以同时显示 Agent 的执行状态。
