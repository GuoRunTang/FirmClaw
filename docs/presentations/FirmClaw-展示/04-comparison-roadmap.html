三、FirmClaw 的独特优势

1. 完全可理解

从零搭建，每个模块都有清晰的注释和文档。不同于 OpenClaw 的庞大代码库，FirmClaw 只有 65 个文件，代码总量可控。

→ 适合作为学习 AI Agent 的教学项目

2. 完全可控

开源 + 本地优先，不依赖任何云服务。可以搭配本地 LLM 实现完全离线运行，适合对数据安全有严格要求的单位。

→ 适合单位内网部署

3. 极简依赖

外部依赖仅 7 个（openai/ajv/ws/dotenv/tsx/typescript/pptxgenjs），不引入复杂的原生依赖。

→ 部署简单，环境一致性好

4. 渐进式架构

6 个阶段逐步叠加功能，每个阶段都经过充分测试。新功能不影响已有功能。

→ 可靠性高，适合生产环境

四、未来研发路线图

v7.0 MCP 协议 + Skills 系统

目标：补齐最重要的两个缺失能力——外部工具集成（MCP）和自定义工作流（Skills）。

MCP 协议：实现 Model Context Protocol，对接 Jira/Slack/飞书/数据库等外部服务。类似 Python 中安装 pip 包后自动获得新能力。

Skills 系统：支持 /review-pr、/deploy-staging 等自定义命令，封装可重复的工作流。

预期时间：4-6 周

v7.1 智能路由 + 定时任务 + HTTP Hooks

多模型智能路由：根据任务复杂度自动选择本地/云端模型（简单任务用本地，复杂任务用云端）。

/loop 定时任务：类似 Claude Code 的 /loop，支持 /loop 5m check deploy status。

HTTP Hooks：工具执行前后可调用 HTTP 端点，与外部系统集成。

预期时间：3-4 周

v7.2 向量搜索 + 增强记忆

向量搜索：引入 embedding 模型（如 bge-m3），在 BM25 基础上增加语义搜索能力。

自动记忆提取：Agent 在对话中自动识别重要信息，写入 MEMORY.md，无需手动管理。

预期时间：3-4 周

v8.0 Agent SDK + 浏览器自动化 + 语音交互

Agent SDK：提供编程接口，允许用户用几行代码创建自定义 Agent。类似 Python 中 from firmclaw import Agent。

浏览器自动化：通过 Puppeteer 集成，Agent 可以操作浏览器——填写表单、截图、提取网页内容。

语音交互：通过 Web Speech API 或 Whisper，支持语音输入和语音输出。

预期时间：6-8 周

v9.0 多模态 + 团队协作 + 移动端

多模态输入：支持图片输入（截图分析、图表理解）、PDF 输入（文档问答）。

团队协作：多用户共享 Agent 会话、共享记忆、角色权限管理。

移动端适配：响应式 Web UI + PWA 支持，在手机上也能使用 Agent。

预期时间：8-10 周

五、总结：FirmClaw 的定位

FirmClaw 不是要替代 OpenClaw 或 Claude Code，而是定位于一个可理解、可控、可定制的本地 Agent 框架。它最适合以下场景：

学习 AI Agent 原理

代码量小、结构清晰、文档完整，是学习 Agent 架构的最佳教材

单位内网部署

搭配本地 LLM，完全离线运行，数据不外泄

定制化需求

可以根据单位的具体需求，修改源码或添加自定义工具

二次开发基础

作为基座，在此基础上开发单位专属的 AI 应用
