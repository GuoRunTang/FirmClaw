项目概述

FirmClaw 是一个从零搭建的本地优先 AI 智能体框架，核心采用 ReAct（Reasoning + Acting）架构。

P1Agent LoopReAct 循环：LLM 思考 → 调工具 → 观察 → 继续思考
P2工具系统极简设计：read / write / edit / bash
P3会话管理JSONL 持久化存储，记忆的骨架
P4系统提示词动态组装 SOUL.md + 工具定义 + 记忆
P5上下文压缩LLM 摘要 + token 裁剪，生存机制
P6记忆系统结构化记忆 + BM25 全文搜索
P7网关层WebSocket + 多客户端 + 子智能体编排

Agent Loop — ReAct 核心解析

Agent Loop 是整个系统的心脏，实现经典的 ReAct 模式

Reason
(思考)

LLM 分析任务
制定行动计划

→

Act
(行动)

调用工具执行
搜索/读文件/命令

→

Observe
(观察)

接收工具结果
作为新上下文

← 循环直到任务完成 →
▸ 自主循环：不断迭代处理不确定性任务
▸ 可解释性：思考过程实时输出
▸ 错误自我修正：根据结果主动调整策略
▸ 安全控制：危险操作需人工审批

上下文管理 — 三级压缩策略

长对话场景下的生存机制

L1优先级
LLM 摘要压缩
50条消息 → ~2000 token
保留语义和关键决策，压缩比 93%
L2优先级
工具结果截断
单条 > 500 token 截断
减少冗余输出，保留关键信息
L3优先级
旧消息移除
整体超限时移除
从最早消息开始，确保窗口不溢出
