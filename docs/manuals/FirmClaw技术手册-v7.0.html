项目简介

FirmClaw 是一个从零搭建的 本地优先 AI 智能体框架，核心采用 ReAct（Reasoning + Acting）架构。灵感来源于 Claude Code / OpenClaw，目标是实现一个可理解、可扩展、本地运行的生产级智能体系统。v7.0 在 v6.0 基础上新增了 Skill 技能系统、MCP 协议扩展和联网搜索三大能力，v7.3 新增了会话问题追踪系统。

MCP 协议扩展 v7.0

源码：src/mcp/ — 通过 Model Context Protocol 连接外部工具服务器，无缝扩展 FirmClaw 的工具能力。MCP 工具自动注册到内置 ToolRegistry，继承全部 5 层安全防护。

MCPClientManager 核心接口

MCPClientManager — src/mcp/mcp-client-manager.ts
class MCPClientManager {
  /** 从 YAML 配置文件加载 MCP Server 定义 */
  loadConfig(configPath: string): Promise<void>;

  /** 自动连接所有 MCP Server 并注册工具 */
  autoConnect(tools: ToolRegistry): Promise<void>;

  /** 连接指定 MCP Server */
  connect(serverName: string): Promise<{ count: number; toolNames: string[] }>;

  /** 断开指定 MCP Server */
  disconnect(serverName: string): Promise<void>;

  /** 将 MCP Server 的工具同步到 ToolRegistry */
  syncToolsToRegistry(serverName: string, tools: ToolRegistry): void;

  /** 获取所有 Server 的连接状态 */
  getStatus(): Array<MCPStatus>;

  /** 判断指定 Server 是否已连接 */
  isConnected(serverName: string): boolean;
}

IssueTracker 核心接口

class IssueTracker {
  constructor(config?: IssueTrackerConfig);

  /** 创建 After Hook 回调（用于 HookManager.registerAfter） */
  createAfterHook(): AfterHookFn;

  /** 绑定到 EventStream（监听 error/context_trimmed/approval_denied/agent_status） */
  bindEvents(events: EventStream): void;

  /** 设置当前会话上下文 */
  setSession(sessionId: string, title?: string): void;

  /** 从自由文本分类并记录问题 */
  recordFromText(text: string): IssueRecord | null;

  /** 获取指定会话的分类统计 */
  getSessionSummary(sessionId: string): CategoryStats[];

  /** 获取全局汇总（跨所有会话） */
  getGlobalSummary(): IssueSummary;

  /** 离线扫描 JSONL 会话文件 */
  async scanSessions(sessionsDir: string): Promise<{ scanned: number; issuesFound: number }>;

  /** 获取底层 Store 引用 */
  getStore(): IssueStore;
}

双通道接入方式

通道接入点捕获内容
通道 A：After HookHookManager.registerAfter('*', ...)所有工具的 isError 结果（bash 错误、文件操作失败等）
通道 B：EventStreamevents.on('error' | 'context_trimmed' | ...)API 错误、上下文裁剪、审批拒绝、智能体状态异常

11 个一级分类

分类代码中文名说明典型触发模式
ENV环境问题Node.js 运行时错误、文件系统异常ENOENT、version mismatch、EACCES
DEP依赖问题Python/npm 依赖缺失、系统工具缺失ModuleNotFoundError、command not found
ENCODING编码问题中文路径乱码、文件编码错误UnicodeDecodeError、garbled
APILLM API 问题API 调用失败、token 超限、重试失败LLM API error、retry failed
TOOL工具执行问题工具崩溃、超时、非零退出码crashed、timed out、exit code
PERM权限问题路径拒绝、命令拦截、文件保护Permission denied、Access denied、blacklist
APPROVAL审批问题人工审批被拒绝/超时approval denied、approval timed out
PARAM参数问题JSON 解析失败、参数校验失败Invalid JSON arguments、Parameter validation
AGENT智能体交互问题死循环、子智能体失败max turns reached、subagent failed
CONTEXT上下文问题Token 裁剪、摘要压缩触发context trimmed、summary generated
HOOK钩子问题Before Hook 拒绝执行Execution denied by hook

规则优先级设计

规则按优先级从高到低排列，先匹配先生效。关键优先级决策：

  • AGENT 规则在 TOOL 之前 — 避免 "subagent failed" 中的 "failed" 被 TOOL 的超时规则误匹配
  • API 规则在最前 — LLM API 错误是最关键的问题，需要优先识别
  • PERM 规则包含通用 "Permission denied" — 需在更具体的黑名单/保护文件规则之后

自定义规则扩展

const classifier = new IssueClassifier();

// 运行时添加自定义规则（插入到默认规则之前，优先匹配）
classifier.addRule({
  pattern: /CustomError: specific pattern/i,
  category: 'TOOL',
  subCategory: 'custom_error',
  severity: 'error',
});

const result = classifier.classify('CustomError: specific pattern occurred');
// → { category: 'TOOL', subCategory: 'custom_error', severity: 'error' }

能力与边界

FirmClaw 能做什么

  • 理解自然语言指令并自主规划执行步骤
  • 执行终端命令、读写文件、联网搜索
  • 通过 Skill 系统注入领域专业知识
  • 通过 MCP 协议接入外部工具服务器
  • 自动追踪和分类运行过程中的所有问题
  • 生成问题分析报告和改进建议
  • 多轮对话 + 长期记忆 + 全文搜索
  • 通过 WebSocket 网关远程交互
  • 完全本地运行，数据不出本机

FirmClaw 不能做什么

  • 不能运行 GUI 程序或操作图形界面
  • 不能直接访问数据库（需通过 bash 或 MCP 工具间接操作）
  • 不支持向量数据库 / RAG（当前使用 BM25 纯文本搜索）
  • 不内置代码执行沙箱（依赖 Node.js 子进程）
  • 子智能体不支持嵌套（子智能体不能再创建子智能体）

5 层安全防护

① JSON Schema 参数校验（ajv）
② Before/After Hook 拦截
③ 路径白名单 + 命令黑名单
④ 人工审批网关（Human-in-the-Loop）
⑤ Prompt Injection 防护（自动标记净化）
