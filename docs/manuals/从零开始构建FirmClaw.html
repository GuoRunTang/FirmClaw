什么是AI智能体（Agent）？

想象一下，你有一个AI助手，它不仅能和你对话，还能：

📖 读取文件

自动读取项目中的代码、文档，理解项目结构

📝 写入文件

创建新文件，添加代码、文档或配置

✏️ 编辑文件

精确修改文件中的特定部分，而不影响其他内容

💻 执行命令

运行终端命令，安装依赖、构建项目、运行测试

这就是AI智能体——它不只是聊天机器人，而是一个能实际完成任务的数字助手。

为什么需要FirmClaw？

1

LLM的局限性

大型语言模型（如GPT、Claude）本身只能"思考"和"对话"，但无法直接操作你的电脑。它们没有"手"来写代码、读文件或执行命令。

2

工具调用（Function Calling）

为了解决这个问题，LLM提供了"工具调用"功能。你可以告诉模型："我有这些工具可用：read_file、write_file、bash"，当模型需要时，它会请求使用这些工具。

3

ReAct架构

FirmClaw实现了ReAct（Reasoning + Acting）架构，让AI智能体能够：思考 → 选择工具 → 执行工具 → 观察结果 → 再思考，循环往复直到完成任务。

ReAct架构：AI智能体的核心

1. 思考

分析用户需求

→

2. 选择工具

决定使用哪个工具

→

3. 执行工具

读取/写入/编辑/命令

→

4. 观察

查看工具执行结果

→

5. 再思考

根据结果调整策略

这个循环会持续进行，直到AI智能体认为任务已经完成。

TypeScript基础

TypeScript是JavaScript的"超集"，添加了类型系统。让我们对比一下：

// JavaScript没有类型检查
function add(a, b) {
return a + b;
}
// 这样调用不会报错，但逻辑错误
const result = add("hello", 5);
// TypeScript有类型检查
function add(a: number, b: number): number {
return a + b;
}
// 这样会报错：参数类型不匹配
const result = add("hello", 5);

TypeScript在编译时就能发现错误，让代码更可靠、更易维护。

TypeScript核心概念

1

接口（Interface）

定义对象的"形状"，告诉TypeScript这个对象应该有哪些属性

interface User {
name: string;
age: number;
email: string;
}
2

类（Class）

面向对象编程的基础，封装数据和操作

class Agent {
private name: string;
constructor(name: string) {
this.name = name;
}
public greet(): void {
console.log(`Hello, I'm {this.name}`);
}
}
3

异步编程（Async/Await）

处理耗时操作（如文件读取、网络请求）的优雅方式

async function loadFile(path: string): Promise<string> {
const content = await fs.readFile(path, 'utf-8');
return content;
}

LLM客户端：连接AI大脑

LLMClient是FirmClaw与大型语言模型通信的桥梁。它负责：

发送请求

将用户消息和工具定义发送给LLM API

接收响应

解析LLM返回的思考和工具调用请求

格式转换

在内部格式和OpenAI格式之间转换

// 简化的LLMClient使用示例
const llm = new LLMClient(apiKey, baseURL, model);
// 发送消息，获取LLM的思考和工具调用请求
const response = await llm.chat([
{ role: 'system', content: '你是一个AI助手...' },
{ role: 'user', content: '帮我创建一个README.md' }
]);
// LLM可能返回：{"tool_calls": [{"name": "write_file", "arguments": {...}}]}

LLMClient隐藏了API调用的细节，让上层代码只关注业务逻辑。

工具系统：AI的"手"

工具是AI智能体实际执行操作的方式。FirmClaw提供了4个核心工具：

read_file

读取文件内容

write_file

写入新文件

edit_file

编辑现有文件

bash

执行终端命令

// 读取文件工具的定义
const readTool = {
name: 'read_file',
description: '读取文件内容',
parameters: {
type: 'object',
properties: {
path: { type: 'string', description: '文件路径' }
},
required: ['path']
},
run: async (path: string) => {
return await fs.readFile(path, 'utf-8');
}
};

每个工具都有名称、描述、参数定义和执行函数。

工具注册中心（ToolRegistry）

ToolRegistry是工具的"管理员"，负责：

注册工具

添加新工具到系统中

验证参数

确保调用工具时传入正确的参数

执行工具

安全地调用工具的执行函数

权限检查

确保工具调用符合安全策略

// 注册工具的示例
const tools = new ToolRegistry();
// 注册4个核心工具
tools.register(readTool);
tools.register(writeTool);
tools.register(editTool);
tools.register(bashTool);
// 执行工具
const result = await tools.execute('read_file', { path: './README.md' });

AgentLoop：智能体的核心大脑

AgentLoop实现了ReAct架构的核心循环。让我们看看它的简化代码：

class AgentLoop {
async run(userMessage: string) {
// 1. 构建上下文（包含历史消息、系统提示词）
const context = await buildContext(userMessage);
// 2. 调用LLM，获取思考和工具调用请求
const llmResponse = await llm.chat(context);
// 3. 解析LLM响应，执行工具
for (const toolCall of llmResponse.tool_calls) {
const result = await tools.execute(toolCall.name, toolCall.arguments);
// 4. 将工具结果添加到上下文
context.push({ role: 'tool', content: result });
}
// 5. 再次调用LLM，生成最终答案
const finalResponse = await llm.chat(context);
return finalResponse;// 返回最终结果
}
}

这个循环会持续进行，直到AI智能体认为任务已经完成。

会话管理：记住对话历史

如果没有会话管理，每次对话都是新的，AI无法记住之前的对话内容。会话管理系统负责：

持久化存储

将对话保存到文件，重启后可以恢复

多会话支持

同时管理多个对话，切换自如

上下文裁剪

自动删除旧消息，避免token消耗过多

摘要压缩

用摘要代替长对话，节省token

// 会话管理的使用示例
// 创建新会话
const session = await sessionManager.create(workDir, userMessage);
// 会话ID: "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
// 恢复会话
await sessionManager.resume('a1b2c3d4-e5f6-7890-abcd-ef1234567890');
// 列出所有会话
const sessions = await sessionManager.listSessions();

为什么这样设计？

1

模块化设计

每个模块职责单一，易于理解和维护。就像搭积木一样，可以单独替换或升级某个模块。

2

类型安全

TypeScript的类型系统在编译时就能发现错误，避免运行时崩溃。

3

可扩展性

可以轻松添加新工具、新功能，而不会影响现有代码。

4

可测试性

每个模块都可以独立测试，确保功能正确。
