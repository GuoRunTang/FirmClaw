项目简介

FirmClaw 是一个从零搭建的 本地优先 AI 智能体框架，核心采用 ReAct（Reasoning + Acting）架构。灵感来源于 OpenClaw，目标是实现一个可理解、可扩展、本地运行的生产级智能体系统。v6.0 在 v5.0 基础上新增了 WebSocket 远程交互能力。

风险等级判定规则

等级触发条件处理方式
CRITICAL删除操作（rm -rf）、系统级命令、写入敏感路径必须人工审批，超时自动拒绝
HIGH写文件、网络请求、环境变量修改需人工审批（可配置自动通过）
MEDIUM读大文件、复杂 bash 管道记录审计日志，自动执行

WebSocket Gateway v5.1

源码：src/gateway/ — 基于 ws 库实现的 JSON-RPC 2.0 远程交互网关，支持多客户端并发连接、Token 认证、会话管理和实时事件推送。

🔌 如何启动 Gateway

Gateway 不是独立的进程，而是在 FirmClaw REPL 中通过斜杠命令启动的：
/serve [port] — 启动（默认端口 3000）  /serve stop — 停止  /serve status — 查看状态
启动后自动生成认证 Token（格式 fc_<hex>），并通过控制台输出 WebSocket 和 Web UI 的连接地址。

JSON-RPC 2.0 类型定义

interface JsonRpcRequest {
  jsonrpc: '2.0';
  id?: string | number;
  method: string;
  params?: any;
}

interface JsonRpcResponse {
  jsonrpc: '2.0';
  id: string | number | null;
  result?: any;
  error?: JsonRpcError;
}

interface JsonRpcError {
  code: number;
  message: string;
  data?: any;
}

interface JsonRpcNotification {
  jsonrpc: '2.0';
  method: string;
  params?: any;
}

安全说明：AuthGuard 使用 crypto.timingSafeEqual() 进行 Token 比较，防止计时攻击。Token 通过 crypto.randomBytes(32) 生成（fc_ 前缀 + 64hex），具备足够的熵值。通过 URL 查询参数传递（?token=xxx），生产环境建议配合 TLS 使用。

子智能体管理器 v5.3

源码：src/agent/subagent-manager.ts、src/tools/subagent.ts — 允许主智能体委派子任务给独立的子智能体执行，支持并发控制和超时保护。

SubagentConfig

interface SubagentConfig {
  task: string;               // 子任务描述（必需）
  allowedTools?: string[];    // 限制可用工具（默认：全部）
  maxTurns?: number;          // 最大循环轮次（默认 5）
  inheritSession?: boolean;   // 是否继承父会话上下文（默认 false）
  timeoutMs?: number;         // 超时时间（默认 120000 = 2min）
}

SubagentResult

interface SubagentResult {
  subagentId: string;    // 子智能体唯一 ID
  task: string;          // 任务描述
  text: string;          // 最终输出文本
  turns: number;         // 实际执行轮次
  toolCalls: number;     // 工具调用次数
  durationMs: number;    // 执行耗时（毫秒）
  timedOut: boolean;     // 是否超时
  error?: string;        // 错误信息（如有）
}

子智能体生命周期

主智能体 (AgentLoop)
  │
  ├─ 调用 subagent 工具 ──────────────────→ SubagentManager.spawn()
  │                                           │
  │                                           ├─ 检查并发限制
  │                                           ├─ 创建独立 AgentLoop
  │                                           ├─ 注册独立 ToolRegistry
  │                                           ├─ 设置超时计时器
  │                                           │
  │                                           │  ┌─ AgentLoop.run(task) ─┐
  │                                           │  │  ReAct 循环           │
  │                                           │  │  maxTurns = 5        │
  │                                           │  └──────────────────────┘
  │                                           │
  │                                           ├─ 收集结果
  │                                           ├─ 清理资源
  │                                           │
  ←─ 返回 SubagentResult ────────────────────┘
  │
  ├─ 将结果注入上下文
  └─ 继续主循环
共享 LLM 连接：子智能体与主智能体共享同一个 LLMClient 实例，复用底层 HTTP 连接池。每个子智能体拥有独立的 ToolRegistry，可通过 allowedTools 限制可用工具集。
超时保护：默认 2 分钟超时，超时后自动终止子智能体并返回 timedOut: true。最大并发数默认为 3，超出时 spawn() 将拒绝创建新子智能体。

WebSocket 安全（v6.0 新增）

  • Token 认证：所有 WebSocket 连接必须携带有效 token（URL 查询参数）
  • Timing-safe 比较：使用 crypto.timingSafeEqual() 防止计时攻击
  • 连接数限制：默认最大 10 个并发连接，超出拒绝
  • 消息大小限制：默认最大 1MB，防止内存耗尽
  • 会话锁定：每个连接同一时间只能运行一个 agent 任务

安全最佳实践

  1. 始终使用默认安全配置：九层防护默认全部启用
  2. 生产环境使用 TLS：WebSocket + HTTP 配合 TLS 加密
  3. 保护 Token：不要在公开渠道分享 Gateway Token
  4. 限制 maxConnections：根据服务器资源调整
  5. 子智能体限制工具：使用 allowedTools 限制子智能体可用工具
  6. 定期导出审计日志：使用 /audit-export 备份操作记录
  7. 检查 PromptGuard 统计：频繁命中可能表示注入攻击
安全设计原则：渐进增强 — 所有安全层均为可选但推荐。最小可用系统仅需 LLMClient + AgentLoop + 工具。生产环境建议启用全部安全层。
