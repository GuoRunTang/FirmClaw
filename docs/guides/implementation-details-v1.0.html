系统提示词（System Prompt）

系统提示词是发送给 LLM 的第一条消息。它定义了智能体的身份、能力和行为规则。LLM 根据"它"来决定是直接回答还是调用工具。

Figure 1 — System Prompt Content (index.ts L28-40)
关键点：系统提示词中明确告诉 LLM "使用 bash 工具执行命令"。如果没有这句话，LLM 可能不知道自己有工具可用，而尝试纯文本回答它无法回答的问题。

工具定义 — 发送给 LLM 的真实数据

源码逐行解读

EventStream 完整源码（仅 49 行）

// 导入 Node.js 内置的 EventEmitter
import { EventEmitter } from 'node:events';

// 限制允许的事件类型 —— 防止拼写错误
export type AgentEventType =
  | 'thinking_delta'   // LLM 生成的文本片段
  | 'tool_start'       // 工具开始执行
  | 'tool_end'         // 工具执行完成
  | 'message_end'      // 最终回复完成
  | 'error';           // 出错了

// 统一的事件对象格式
export interface AgentEvent {
  type: AgentEventType;  // 事件类型
  data?: unknown;         // 事件数据
}

// 继承 EventEmitter，只做了两件小事：
export class EventStream extends EventEmitter {

  // ① 重写 emit：自动把 data 包装成 { type, data } 对象
  emit(event: AgentEventType, data?: unknown): boolean {
    return super.emit(event, { type: event, data });
    //           ^^^^^^^^^^^^^^^^^^^^^^^^
    //  callback 收到的始终是 { type: "tool_start", data: {...} }
  }

  // ② 重写 on：callback 参数类型化
  on(event: AgentEventType, listener: (event: AgentEvent) => void): this {
    return super.on(event, listener);
  }
}

Agent Loop 中的 emit 调用点（标注）

async run(userMessage: string): Promise<AgentResult> {
  while (turns < this.config.maxTurns) {
    turns++;

    // ═══ Step 1: 调用 LLM ═══
    const response = await this.llm.chat(messages, this.tools,
      (delta) => {
        this.events.emit('thinking_delta', delta);  // ← 事件 1: 每个 chunk
      },
    );

    // ═══ Step 2: 判断响应类型 ═══
    if (!response.tool_calls?.length) {
      this.events.emit('message_end', response.content);  // ← 事件 5: 最终答案
      return { text: response.content, turns, toolCalls: totalToolCalls };
    }

    // ═══ Step 3: 执行工具 ═══
    for (const toolCall of response.tool_calls) {
      try {
        toolArgs = JSON.parse(toolCall.function.arguments);
      } catch {
        this.events.emit('error', `Failed to parse...`);     // ← 事件 6: JSON 解析失败
        continue;
      }

      this.events.emit('tool_start', { toolName, args: toolArgs });  // ← 事件 2: 开始执行

      const tool = this.tools.get(toolName);
      if (!tool) {
        this.events.emit('error', `Unknown tool: "${toolName}"`);   // ← 事件 6: 未知工具
        continue;
      }

      try {
        const result = await tool.execute(toolArgs);
        this.events.emit('tool_end', { toolName, result: result.content }); // ← 事件 3: 执行完成
      } catch (error) {
        this.events.emit('error', `Tool crashed: ${error}`);           // ← 事件 6: 工具崩溃
      }
    }
  }
}

CLI 中的 on 订阅（标注）

// index.ts — CLI 订阅所有事件，展示给用户

const events = agent.getEvents();

// 订阅 thinking_delta → LLM 正在生成文字
events.on('thinking_delta', (e) => {
  process.stdout.write(e.data as string);  // 直接输出到终端，不打换行
});

// 订阅 tool_start → 工具开始执行
events.on('tool_start', (e) => {
  const data = e.data as { toolName: string; args: Record<string, unknown> };
  console.log(`\n>>> [${data.toolName}] ${JSON.stringify(data.args)}`);
});

// 订阅 tool_end → 工具执行完成
events.on('tool_end', (e) => {
  const data = e.data as { result: string };
  const preview = data.result.length > 300
    ? data.result.substring(0, 300) + '...(truncated)'
    : data.result;
  console.log(`<<< [${data.toolName}] ${preview}`);
});

// 订阅 error → 出错了
events.on('error', (e) => {
  console.error(`\n[Error] ${e.data}`);
});
总结：EventStream 的本质就是一个 中间人。Agent Loop 只管"喊话"（emit），CLI / Web UI / 日志 各自"接听"（on）。两边互不依赖，想换显示方式只需改订阅逻辑，Agent Loop 完全不用动。
