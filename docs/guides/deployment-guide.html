1
安装 Node.js
~5 分钟

下载安装包

访问 https://nodejs.org，下载 LTS (长期支持版)：

  • 推荐版本：20.x LTS 或 22.x LTS
  • 文件名形如 node-v20.x.x-x64.msi
  • 选择 .msi 安装包（Windows 原生安装器）

安装步骤

  • 双击 .msi 文件，一路 Next
  • 勾选 "Automatically install the necessary tools"（会自动安装 chocolatey 和构建工具）
  • 默认安装路径即可：C:\Program Files\nodejs
  • 安装完成后，新开一个 PowerShell 窗口（旧窗口的环境变量不会刷新）

验证安装

powershellnode --version      # 应显示 v20.x.x 或 v22.x.x
npm --version       # 应显示 10.x.x
✓ Node.js 和 npm 会自动加入 PATH，你无需手动配置环境变量。

└── dist/           # tsc 会重建

方式 C：压缩包

# 源电脑上，排除 node_modules 后压缩
# 然后通过网络传输到新电脑

3
安装项目依赖
~1-3 分钟

打开 PowerShell，进入项目目录：

cd D:\FirmClaw    # 替换为你实际的路径

# 安装所有依赖
npm install

npm install 会读取 package-lock.json，安装以下 5 个依赖包：

包名版本作用
openai^6.32.0OpenAI SDK（OpenAI 兼容格式）
dotenv^17.3.1加载 .env 环境变量
tsx^4.21.0TypeScript 直接运行器（无需编译）
typescript^6.0.2TypeScript 编译器
@types/node^25.5.0Node.js 类型定义
✓ 安装完成后，项目目录会多出一个 node_modules/ 文件夹（~150MB）。

如果 npm install 报错

# 清除 npm 缓存后重试
npm cache clean --force
npm install
4
配置环境变量 (.env)
~1 分钟

在项目根目录创建 .env 文件。如果已经从源电脑复制过来，检查并更新 API Key 即可。

如果没有，新建文件：

# 方法 1：用记事本创建
notepad D:\FirmClaw\.env

# 方法 2：PowerShell 创建
New-Item -Path D:\FirmClaw\.env -ItemType File

写入以下内容（替换为你自己的 Key）：

# ═══ FirmClaw 环境变量配置 ═══

# LLM 提供商配置（默认 MiniMax M2.7）
# 也支持 ANTHROPIC_AUTH_TOKEN / ANTHROPIC_BASE_URL / ANTHROPIC_MODEL 环境变量
LLM_API_KEY=你的MiniMax密钥
LLM_BASE_URL=https://api.minimax.chat/v1
LLM_MODEL=MiniMax-M2.7
关于 .env 与系统环境变量
dotenv 加载规则：.env 不会覆盖已存在的系统环境变量。
优先级：系统环境变量 > .env 文件 > 代码默认值。
所以如果新电脑的系统环境变量中设了同名变量，.env 的值会被忽略。
5
验证安装
~2 分钟

依次运行 3 个测试，确认每个模块工作正常：

测试 1：Bash 工具（无需网络）

npm run test:bash

预期输出：3 项测试全部 PASS（文件操作、命令执行、超时控制）。

测试 2：LLM 连通性（需要网络 + API Key）

npm run test:llm

预期输出：MiniMax API 返回普通文本回复 + 工具调用响应。

测试 3：完整 Agent 循环

npm run test:agent

预期输出：LLM 调用 bash 工具 → 获取结果 → 给出最终答案，完整 ReAct 循环。

✓ 3 个测试全部通过 = 部署成功！
测试 2/3 失败的常见原因：
• .env 文件位置不对（必须在项目根目录）
• API Key 无效或过期
• 网络无法访问 api.minimax.chat

安装流程总览
常见问题排查

Q1: node 或 npm 不是内部命令

Node.js 安装后 PATH 未生效。解决：

  • 关闭所有 PowerShell 窗口，重新打开
  • 或检查环境变量：echo $env:PATH 是否包含 Node.js 路径
  • 如果还不行，手动添加：C:\Program Files\nodejs 到系统 PATH

Q2: npm install 报权限错误

# 方法 1：以管理员身份运行 PowerShell
# 方法 2：清除 npm 缓存
npm cache clean --force
npm install

# 方法 3：如果有 .npmrc 权限问题
npm config set userconfig $env:USERPROFILE\.npmrc

Q3: LLM_API_KEY is not set

原因：.env 文件不存在、位置不对、或变量名拼写错误。 
# 确认 .env 在项目根目录
Test-Path D:\FirmClaw\.env   # 应返回 True

# 检查内容
Get-Content D:\FirmClaw\.env

Q4: 测试 LLM 超时或 401 错误

  • 401 Unauthorized：API Key 无效或过期，去对应平台重新生成
  • 超时：检查网络是否能访问 api.minimax.chat
  • 429 Too Many Requests：请求频率过高，等几秒重试

Q5: 如何换用其他 LLM 提供商？

项目使用 OpenAI 兼容格式，只需修改 .env：

# 示例：切换到 Kimi
LLM_API_KEY=你的Kimi密钥
LLM_BASE_URL=https://api.moonshot.ai/v1
LLM_MODEL=moonshot-v1-8k

# 示例：切换到 DeepSeek
LLM_API_KEY=你的DeepSeek密钥
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat

Q6: TypeScript 编译错误

# 如果直接运行报 TS 错误，清理并重新安装
Remove-Item -Recurse -Force node_modules
npm install

# 项目使用 tsx 直接运行 TS，一般不需要手动编译
# 如果需要编译为 JS：
npm run build    # 输出到 dist/
