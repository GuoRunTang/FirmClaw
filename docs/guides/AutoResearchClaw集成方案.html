§ 02

AutoResearchClaw 23 阶段流水线

一条从研究想法到会议论文的端到端自动化流水线，分为 8 个阶段，涵盖文献发现、知识综合、实验设计、论文写作与最终化。

Phase A · 研究定位

定义研究问题、范围与约束，识别关键变量与依赖关系

Phase B · 文献发现

系统搜索、筛选与结构化知识提取，建立研究背景

Phase C · 知识综合

主题聚类与研究空白分析，生成可验证假设

Phase D · 实验设计

设计实验协议，生成可执行代码，规划计算资源

Phase E · 实验执行

沙盒环境运行实验，迭代优化直到结果达标

Phase F · 分析决策

统计分析与决策判断，决定进入写作或回滚重做

Phase G · 论文写作

数据驱动撰写论文，模拟同行评审并修订

Phase H · 最终化

质量门控、引用验证与论文导出发布

§ 03

23 阶段逐阶段详解

每个阶段的输入、LLM 调用方式、产出物与技术实现要点。门控阶段需人工审批，失败自动回滚。

Phase A · 研究定位
01 TOPIC_INIT
02 PROBLEM_DECOMPOSE

Stage 01 — TOPIC_INIT

输入：用户 research topic 字符串

LLM：生成 SMART Goal 格式研究目标

输出：goal.md, hardware_profile.json

Stage 02 — PROBLEM_DECOMPOSE

输入：goal.md

LLM：递归分解为独立变量，识别依赖

输出：problem_tree.md

Phase B · 文献发现
03 SEARCH_STRATEGY
04 LITERATURE_COLLECT
05 LITERATURE_SCREEN
06 KNOWLEDGE_EXTRACT

Stage 03-04

多角度查询扩展，并行 web_search 调用，arXiv/Google Scholar 去重

Stage 05 — LITERATURE_SCREEN ⬡ 门控

双重 LLM 筛选（相关性 + 质量），人工审批入口。失败→回滚 Stage 04

Stage 06 — KNOWLEDGE_EXTRACT

批量 LLM 调用，提取：方法名、公式、数据集、结论 → knowledge_cards/

Phase C · 知识综合
07 SYNTHESIS
08 HYPOTHESIS_GEN

Stage 07 — SYNTHESIS

主题聚类 + 研究空白分析 → synthesis.md。识别"已知/未知边界"

Stage 08 — HYPOTHESIS_GEN

生成可操作化假设，每个假设含 H1/H2/H3，指定评估指标

Phase D · 实验设计
09 EXPERIMENT_DESIGN
10 CODE_GENERATION
11 RESOURCE_PLANNING

Stage 09 — EXPERIMENT_DESIGN ⬡ 门控

设计数据集、baseline、评估指标、训练/验证/测试划分 → exp_plan.yaml。失败→回滚 Stage 08

Stage 10 — CODE_GENERATION

核心：生成 models.py/data.py/main.py + validate_code() AST 检查。开启 OpenCode Beast Mode 复杂代码自动委托

Stage 11 — RESOURCE_PLANNING

GPU 分配、任务并行化、依赖图 → schedule.json

Phase E · 实验执行
12 EXPERIMENT_RUN
13 ITERATIVE_REFINE ↺

Stage 12 — EXPERIMENT_RUN

sandbox（.venv）/ docker / simulated 三种模式。stdout 解析 metric_name: value 格式指标。results/ 目录产出

Stage 13 — ITERATIVE_REFINE ↺

循环：编辑代码 → 运行 → 评估 → 判断是否达标（最多 max_iterations 次）。未达标触发 Stage 15 的 PIVOT/REFINE 决策

Phase F · 分析与决策
14 RESULT_ANALYSIS
15 RESEARCH_DECISION

Stage 14 — RESULT_ANALYSIS

统计分析 → experiment_summary.json + results_table.tex + charts/。生成 matplotlib 图表

Stage 15 — RESEARCH_DECISION

PROCEED → Stage 16写论文 | PIVOT → Stage 08重新假设 | REFINE → Stage 13重新实验。最多 PIVOT 2次

Phase G · 论文写作
16 PAPER_OUTLINE
17 PAPER_DRAFT
18 PEER_REVIEW
19 PAPER_REVISION

Stage 16-17

数据驱动：所有指标必须来自 experiment_summary.json，禁止捏造。生成 paper.tex（LaTeX）+ paper_final.md（Markdown）

Stage 18 — PEER_REVIEW

多轮（默认2轮）模拟同行评审，7维度评分，检测"AI Slop"（模板化无实质内容段落）

Stage 19 — PAPER_REVISION

逐条对应评审意见修订，检测实验结果与论文叙述矛盾，输出修订版 paper.tex

Phase H · 最终化
20 QUALITY_GATE
21 KNOWLEDGE_ARCHIVE
22 EXPORT_PUBLISH
23 CITATION_VERIFY ✗

Stage 20 — QUALITY_GATE ⬡ 门控

格式合规 / 引用完整 / 结果一致 / 幻觉检测。失败→回滚 Stage 16

Stage 21-22

知识归档（跨 run 经验积累 MetaClaw），导出 PDF（XeLaTeX）+ 代码仓库

Stage 23 — CITATION_VERIFY ✗ 安全门

逐条验证引用准确性，拦截幻觉引用（BibTeX 存在但正文未用）。发现虚假引用→阻止导出

§ 05

新增研究工具设计

7 个专业化 TypeScript 工具，封装 LLM 调用逻辑，通过 ToolRegistry 注册到 AgentLoop。点击展开查看完整接口定义与代码。

📚
research_literature_search — 文献搜索 ▼

封装 web_search/web_fetch，提供学术论文搜索能力。支持 arXiv、Semantic Scholar 等多数据源，自动去重与相关性排序。

输入接口

interface LiteratureSearchInput { query: string; // 搜索词 max_results?: number; // 默认 10 source?: 'arxiv' | 'semantic_scholar' | 'all'; domain?: string; // ml / cv / nlp / robotics }

输出结构

interface LiteratureSearchOutput { papers: Arraytitle, authors, venue, year, abstract, url}>; search_metadata: { query_used, total_found, sources_queried }; }
⚡
research_experiment_runner — 实验执行引擎 ▼

封装 bash 工具，在 .venv 环境中执行 Python 代码。自动解析 stdout 中的 metric_name: value 格式指标。

输入接口

interface ExperimentRunnerInput { code: string; // Python 代码 python_path?: string; // 默认 .venv/bin/python dependencies?: string[]; // 额外依赖 timeout_sec?: number; // 默认 300 gpu_required?: boolean; work_dir?: string; }

解析格式

// 标准输出格式（stdout 自动解析） accuracy: 0.945 AUC-ROC: 0.9231 AUPRO: 0.8712 // 多条件统计格式 SUMMARY condition=baseline metric=AUC-ROC mean=0.891 std=0.012 SUMMARY condition=proposed metric=AUC-ROC mean=0.923 std=0.008
💻
research_code_generator — 实验代码生成器 ▼

根据方法描述生成完整可执行 Python 实验代码。内置 AST 语法验证、import 依赖检查、安全扫描（禁止 os.system/eval 等危险操作）。支持 PyTorch/TensorFlow/JAX 模板。

输入接口

interface CodeGeneratorInput { description: string; language: 'python' | 'typescript' | 'cpp'; requirements?: string[]; template?: 'pytorch' | 'tensorflow' | 'jax' | 'none'; complexity: 'low' | 'medium' | 'high'; }
📝
research_paper_writer — 论文写作引擎 ▼

支持 outline / section / full / revision 四种模式。数据驱动写作：所有指标必须来自实验结果，禁止捏造数值。自动按目标会议格式输出 LaTeX。

输入接口

interface PaperWriterInput { mode: 'outline' | 'section' | 'full' | 'revision'; topic: string; existing_paper?: string; // revision 模式 review_feedback?: string; // revision 模式 target_venue?: string; // neurips / cvpr / icml data_results?: object; // 实验数据注入 }
🔍
research_peer_reviewer — 模拟同行评审 ▼

多轮（默认2轮）模拟同行评审，输出 7 维度评分 + 逐条详细意见 + 最终建议（accept / minor_revision / major_revision / reject）。自动检测"AI Slop"段落。

输出结构

interface PeerReviewerOutput { reviews: Arrayround: number; overall_score: number; // 1-10 scores: { novelty, technical_quality, clarity, significance }; strengths: string[]; weaknesses: string[]; detailed_comments: string[]; recommendation: 'accept' | 'minor_revision' | 'major_revision' | 'reject'; }>>; quality_score: number; }
📊
research_result_analyzer — 实验结果分析器 ▼

从实验 stdout 输出自动生成 experiment_summary.json、LaTeX 表格、图表描述（matplotlib 代码）。支持与假设对照，自动判断假设是否被支持。

输出结构

interface ResultAnalyzerOutput { summary: { primary_metric, proposed_method_score, improvement_over_baseline? }; tables: Arraycaption, format: 'latex'|'markdown', content }>>; charts: Arraytype, data_description }>>; hypotheses_evaluation: Arrayhypothesis, supported: boolean, evidence }>>; }
✓
research_citation_verifier — 引用真实性验证 ▼

关键安全门：逐条验证论文中引用的准确性。识别：未使用引用（unused）、缺失引用（missing）、幻觉引用（hallucinated，BibTeX 存在但正文未引用）。发现虚假引用→阻止导出。

输出结构

interface CitationVerifierOutput { verified: boolean; issues: Arraytype: 'unused' | 'missing' | 'mismatch' | 'hallucinated'; bibtex_key: string; description: string; }>>; statistics: { total_references, used_in_text, unused, hallucinated }; cleaned_references?: object[]; // 移除问题引用后 }

§ 08

技术风险与缓解

风险类型 描述 缓解方案 等级
LLM 生成虚假数据 论文写作时 LLM 可能捏造指标数值 强制要求 experiment_summary.json 作为唯一数据源，表格数值禁止手写 高
实验执行资源消耗 并行跑多个实验导致 GPU 显存溢出 Stage 11 资源调度器控制并发数，最大 2 个实验并行 中
代码生成质量参差 生成的代码语法错误或逻辑有问题 AST 语法验证 + sandbox 执行测试 + validate_code() 中
引用幻觉 论文中出现不存在的引用 Stage 23 citation_verifier 作为强制安全门，虚假引用→阻止导出 高
API 成本过高 23 阶段每阶段多次 LLM 调用，成本快速累积 上下文压缩 + 分阶段结果缓存 + 减少无意义重试 中
论文格式不一致 不同会议格式差异大，模板适配复杂 模板化 + 分会议配置文件（neurips.yaml / cvpr.yaml 等） 低

核心设计原则

🛡️ 数据驱动写作

所有论文指标必须来自真实实验结果，禁止捏造。表格数值直接注入，无中间层篡改可能。

🔒 质量门控强制拦截

引用验证（Stage 23）和质量门控（Stage 20）作为不可绕过的安全门，失败必须修复才能继续。

🧩 模块化工具

每个研究能力独立可复用、可测试、可替换。工具之间通过结构化数据（JSON）传递实验结果。
