1. 架构总览

FirmClaw 采用 ReAct（Reasoning + Acting）架构。LLM 在推理过程中会自主决定调用哪个工具，工具的返回结果作为 Observation 反馈给 LLM，驱动下一轮推理。

ReAct 循环 × 工具调用流程
用户输入用户消息 → AgentLoop
LLM 推理选择工具 → 生成参数
参数校验AJV JSON Schema 校验
Before Hooks可修改参数 / 拒绝执行
权限检查PermissionPolicy 中间件
执行工具tool.execute(params, ctx)
After Hooks审计日志 / 处理结果
PromptGuard扫描注入攻击

文件结构：

text
src/tools/
├── types.ts          # Tool 接口定义（你必须实现的）
├── context.ts        # ToolContext（运行时注入的工作目录等）
├── registry.ts       # ToolRegistry（注册中心 + 执行管道）
├── permissions.ts    # 权限策略（路径白名单 + 命令黑名单）
├── hook-manager.ts   # Before/After 钩子
├── bash.ts           # 内置：Shell 命令执行
├── read.ts           # 内置：文件读取
├── write.ts          # 内置：文件写入
├── edit.ts           # 内置：文件查找替换
└── subagent.ts       # 内置：子智能体

src/index.ts          # ← 在这里注册你的工具

2. 工具接口定义

每个工具必须实现 Tool 接口，共 4 个字段：

typescript
import type { Tool, ToolResult, ToolContext } from './tools/types.js';

export const myTool: Tool = {
  // ① 名称 —— LLM 通过这个名字调用工具（必须唯一）
  name: 'my_tool',

  // ② 描述 —— 告诉 LLM 这个工具能做什么（直接影响 LLM 的调用决策）
  description: '调用公司的 XX 系统执行 YY 操作。传入参数说明...',

  // ③ 参数定义 —— JSON Schema 格式，LLM 据此生成正确的调用参数
  parameters: {
    type: 'object',
    properties: {
      query: {
        type: 'string',
        description: '查询内容',
      },
      limit: {
        type: 'number',
        description: '返回条数上限，默认 10',
      },
    },
    required: ['query'],   // 必填参数列表
  },

  // ④ 执行函数 —— 工具被调用时实际运行的逻辑
  async execute(
    params: Record<string, unknown>,   // LLM 传入的参数
    context: ToolContext,                    // 运行时上下文
  ): Promise<ToolResult> {
    // ... 你的业务逻辑 ...
    return {
      content: '执行结果文本',  // 返回给 LLM 的 observation
      isError: false,            // 可选：标记为错误
    };
  },
};

ToolContext 运行时上下文

字段类型说明
workDirstring工作目录（文件操作的根路径）
sessionIdstring?当前会话 ID（多轮对话时可用）

ToolResult 返回值

字段类型说明
contentstring返回给 LLM 的文本（会作为 observation）
isErrorboolean?标记为错误（LLM 会调整策略重试）
关键原则 description 字段极其重要——LLM 完全依赖它来判断何时、如何调用工具。
好的描述应包含：功能 + 参数说明 + 使用场景 + 注意事项。

3. 工具执行生命周期

当 LLM 决定调用工具时，ToolRegistry.execute() 会依次执行以下 7 个步骤：

STEP 1JSON Schema 校验
→
STEP 2Before Hooks
→
STEP 3权限检查
→
STEP 4执行工具
→
STEP 5After Hooks
→
STEP 6PromptGuard
步骤说明你的工具需要关心？
1. 参数校验AJV 根据 parameters 定义自动校验自动 只需正确定义 schema
2. Before Hooks可在执行前修改参数或拒绝自动 除非你注册了 hook
3. 权限检查路径白名单 + 命令黑名单自动 自定义工具默认放行
4. 执行工具调用你的 execute() 函数是 你的核心逻辑在这里
5. After Hooks审计日志、处理结果自动
6. PromptGuard扫描结果中的注入攻击自动

4. 5 分钟上手：最小化工具

创建文件 src/tools/company-api.ts：

typescript
// src/tools/company-api.ts
import type { Tool, ToolResult } from './types.js';

export const companyApiTool: Tool = {
  name: 'company_api',
  description: '调用公司内部 API。支持 GET/POST 请求。',
  parameters: {
    type: 'object',
    properties: {
      method: {
        type: 'string',
        description: 'HTTP 方法：GET 或 POST',
        enum: ['GET', 'POST'],
      },
      url: {
        type: 'string',
        description: 'API 端点 URL（相对于 BASE_URL）',
      },
      body: {
        type: 'string',
        description: 'POST 请求的 JSON body（可选）',
      },
    },
    required: ['method', 'url'],
  },

  async execute(params): Promise<ToolResult> {
    const { method, url, body } = params as {
      method: string;
      url: string;
      body?: string;
    };

    const BASE_URL = process.env.COMPANY_API_URL || 'http://internal-api.company.com';
    const API_KEY = process.env.COMPANY_API_KEY || '';

    try {
      const response = await fetch(`${BASE_URL}${url}`, {
        method,
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${API_KEY}`,
        },
        body: method === 'POST' ? body : undefined,
      });

      if (!response.ok) {
        return {
          content: `API Error: ${response.status} ${response.statusText}`,
          isError: true,
        };
      }

      const data = await response.json();
      // 截断过大的响应，避免消耗过多 context window
      const resultStr = JSON.stringify(data, null, 2);
      const MAX_LEN = 10000;
      if (resultStr.length > MAX_LEN) {
        return {
          content: resultStr.substring(0, MAX_LEN)
            + `\n\n[Truncated: total ${resultStr.length} chars]`,
        };
      }

      return { content: resultStr };
    } catch (err) {
      return {
        content: `Request failed: ${err instanceof Error ? err.message : String(err)}`,
        isError: true,
      };
    }
  },
};
完成！ 你已经创建了第一个工具。接下来只需在 src/index.ts 中注册它（见第 6 节）。

5. 实战案例

案例 A：HTTP API 调用（REST 服务）

适用于公司有 RESTful API 的场景（ERP、CRM、OA 等）。

typescript
// src/tools/erp-tool.ts
import type { Tool, ToolResult } from './types.js';

export const erpTool: Tool = {
  name: 'erp_query',
  description: `查询公司 ERP 系统数据。
支持模块：
- 订单查询：传入 action="query_orders", 可按 customer_id、status、date_range 筛选
- 库存查询：传入 action="query_inventory", 可按 sku、warehouse 筛选
- 客户信息：传入 action="get_customer", 传入 customer_id
返回 JSON 格式的数据，最多 20 条记录。`.trim(),
  parameters: {
    type: 'object',
    properties: {
      action: {
        type: 'string',
        description: '操作类型：query_orders | query_inventory | get_customer',
        enum: ['query_orders', 'query_inventory', 'get_customer'],
      },
      customer_id: { type: 'string', description: '客户 ID（查询订单时可选）' },
      status: { type: 'string', description: '订单状态（如 pending/shipped/completed）' },
      sku: { type: 'string', description: 'SKU 编号（库存查询时使用）' },
      date_start: { type: 'string', description: '起始日期 YYYY-MM-DD' },
      date_end: { type: 'string', description: '截止日期 YYYY-MM-DD' },
    },
    required: ['action'],
  },

  async execute(params): Promise<ToolResult> {
    const { action, ...filters } = params;
    const BASE = process.env.ERP_API_URL || 'http://erp.internal.company.com/api/v1';

    // 将业务参数映射为 API 查询字符串
    const qs = new URLSearchParams(
      Object.entries(filters).filter(([, v]) => v != null) as [string, string][]
    );

    try {
      const resp = await fetch(`${BASE}/${action}?${qs}`, {
        headers: {
          'Authorization': `Bearer ${process.env.ERP_API_KEY}`,
        },
      });

      if (!resp.ok) {
        return { content: `ERP API error: ${resp.status}`, isError: true };
      }

      const data = await resp.json();
      return { content: JSON.stringify(data, null, 2) };
    } catch (err) {
      return {
        content: `ERP connection failed: ${err instanceof Error ? err.message : String(err)}`,
        isError: true,
      };
    }
  },
};

案例 B：数据库查询

适用于需要直接查询公司数据库的场景。建议使用只读连接。

typescript
// src/tools/db-query.ts
import type { Tool, ToolResult } from './types.js';

export const dbQueryTool: Tool = {
  name: 'db_query',
  description: `查询公司 MySQL/PostgreSQL 数据库。
⚠️ 仅允许 SELECT 查询，禁止 INSERT/UPDATE/DELETE。
使用示例：查询某用户的最近订单、统计销售额等。`.trim(),
  parameters: {
    type: 'object',
    properties: {
      sql: {
        type: 'string',
        description: 'SQL SELECT 查询语句（只允许 SELECT）',
      },
      database: {
        type: 'string',
        description: '目标数据库名（默认 company_main）',
        default: 'company_main',
      },
    },
    required: ['sql'],
  },

  async execute(params): Promise<ToolResult> {
    const sql = (params.sql as string).trim();
    const database = (params.database as string) || 'company_main';

    // 安全检查：只允许 SELECT
    if (!/^SELECT\s/i.test(sql)) {
      return {
        content: 'Error: Only SELECT queries are allowed. Write/modify operations are prohibited.',
        isError: true,
      };
    }

    // 防止超大批量查询
    if (/LIMIT\s/i.test(sql)) {
      const limitMatch = sql.match(/LIMIT\s+(\d+)/i);
      if (limitMatch && parseInt(limitMatch[1]) > 200) {
        return { content: 'Error: LIMIT cannot exceed 200 rows.', isError: true };
      }
    }

    try {
      // Node.js 22+ 原生支持 mysql 或使用 npm 包
      const mysql = await import('mysql2/promise');
      const pool = await mysql.createPool({
        host: process.env.DB_HOST || 'localhost',
        port: parseInt(process.env.DB_PORT || '3306'),
        user: process.env.DB_USER || 'readonly_user',
        password: process.env.DB_PASS || '',
        database,
        connectionLimit: 1,
        // 设置查询超时
        acquireTimeout: 10000,
      });

      const [rows] = await pool.execute(sql);
      await pool.end();

      // 格式化输出
      const result = {
        columns: Object.keys(rows[0] || {}),
        rowCount: Array.isArray(rows) ? rows.length : 0,
        data: rows,
      };

      return { content: JSON.stringify(result, null, 2) };
    } catch (err) {
      return {
        content: `Database error: ${err instanceof Error ? err.message : String(err)}`,
        isError: true,
      };
    }
  },
};

案例 C：文件系统桥接（共享目录）

适用于公司有共享文件目录（SMB/NFS）、需要读取 Excel/CSV/日志等文件的场景。

typescript
// src/tools/file-bridge.ts
import fs from 'node:fs';
import path from 'node:path';
import type { Tool, ToolResult, ToolContext } from './types.js';

export const fileBridgeTool: Tool = {
  name: 'file_bridge',
  description: `读取公司共享目录中的文件。
支持 CSV、JSON、TXT、日志文件。
⚠️ 只读操作，不能写入或删除共享目录中的文件。`.trim(),
  parameters: {
    type: 'object',
    properties: {
      action: {
        type: 'string',
        description: '操作类型：list（列出目录）| read（读取文件）| search（搜索文件名）',
        enum: ['list', 'read', 'search'],
      },
      dir_path: {
        type: 'string',
        description: '共享目录中的相对路径（如 /sales/reports/2024）',
      },
      file_path: {
        type: 'string',
        description: '要读取的文件相对路径（read 时必填）',
      },
      pattern: {
        type: 'string',
        description: '文件名搜索模式，支持通配符（search 时使用，如 *.csv）',
      },
      encoding: {
        type: 'string',
        description: '文件编码（默认 utf-8，CSV 可用 gbk）',
        default: 'utf-8',
      },
    },
    required: ['action'],
  },

  async execute(params, context: ToolContext): Promise<ToolResult> {
    const SHARED_ROOT = process.env.SHARED_DIR || '//fileserver/company';
    const encoding = (params.encoding as string) || 'utf-8';

    try {
      switch (params.action) {
        case 'list': {
          const dir = path.join(SHARED_ROOT, (params.dir_path as string || '').replace(/^\//, ''));
          if (!fs.existsSync(dir)) {
            return { content: `Directory not found: ${dir}`, isError: true };
          }
          const entries = fs.readdirSync(dir)
            .filter(f => !f.startsWith('~') && !f.startsWith('.'))
            .slice(0, 50);
          return { content: JSON.stringify(entries, null, 2) };
        }

        case 'read': {
          const filePath = path.join(SHARED_ROOT, (params.file_path as string || '').replace(/^\//, ''));
          if (!fs.existsSync(filePath)) {
            return { content: `File not found: ${filePath}`, isError: true };
          }
          // 限制文件大小 5MB
          const stat = fs.statSync(filePath);
          if (stat.size > 5 * 1024 * 1024) {
            return { content: `File too large: ${(stat.size / 1024 / 1024).toFixed(1)}MB`, isError: true };
          }
          const content = fs.readFileSync(filePath, encoding);
          // 截断过长内容
          if (content.length > 20000) {
            return { content: content.substring(0, 20000) + '\n... [truncated]' };
          }
          return { content };
        }

        case 'search': {
          const dir = path.join(SHARED_ROOT, (params.dir_path as string || '').replace(/^\//, ''));
          const pattern = (params.pattern as string || '*');
          // 简单搜索实现（生产环境建议用 glob）
          const results: string[] = [];
          function walk(d: string) {
            for (const entry of fs.readdirSync(d, { withFileTypes: true })) {
              const full = path.join(d, entry.name);
              if (entry.isDirectory()) walk(full);
              else if (entry.name.match(pattern.replace(/\*/g, '.*'))) {
                results.push(full.replace(SHARED_ROOT, ''));
              }
            }
          }
          walk(dir);
          return { content: JSON.stringify(results.slice(0, 50), null, 2) };
        }

        default:
          return { content: `Unknown action: ${params.action}`, isError: true };
      }
    } catch (err) {
      return {
        content: `File operation failed: ${err instanceof Error ? err.message : String(err)}`,
        isError: true,
      };
    }
  },
};

案例 D：消息队列操作

typescript
// src/tools/mq-tool.ts
import type { Tool, ToolResult } from './types.js';

export const mqTool: Tool = {
  name: 'message_queue',
  description: `与公司消息队列（RabbitMQ）交互。
支持发送消息到指定队列和查看队列状态。`.trim(),
  parameters: {
    type: 'object',
    properties: {
      action: {
        type: 'string',
        description: '操作：send（发送消息）| status（查看队列状态）',
        enum: ['send', 'status'],
      },
      queue: { type: 'string', description: '队列名称' },
      message: { type: 'string', description: '消息内容（send 时必填）' },
    },
    required: ['action'],
  },

  async execute(params): Promise<ToolResult> {
    const MQ_URL = process.env.RABBITMQ_URL || 'amqp://localhost:5672';

    try {
      switch (params.action) {
        case 'send': {
          if (!params.queue || !params.message) {
            return { content: 'Error: queue and message are required for send action.', isError: true };
          }
          // 使用 amqplib 发送消息
          const amqp = await import('amqplib/callback_api.js');
          const conn = await amqp.connect(MQ_URL);
          const channel = await conn.createChannel();
          await channel.assertQueue(params.queue as string, { durable: true });
          channel.sendToQueue(params.queue as string, Buffer.from(params.message as string), {
            persistent: true,
          });
          await new Promise(r => setTimeout(r, 500));
          await conn.close();
          return { content: `Message sent to queue "${params.queue}" successfully.` };
        }

        case 'status': {
          // 简化示例：返回预定义的队列列表
          const queues = ['order.created', 'order.shipped', 'notification.email', 'report.daily'];
          return { content: JSON.stringify({ queues, mqUrl: MQ_URL }, null, 2) };
        }

        default:
          return { content: 'Unknown action.', isError: true };
      }
    } catch (err) {
      return {
        content: `MQ error: ${err instanceof Error ? err.message : String(err)}`,
        isError: true,
      };
    }
  },
};

6. 注册工具到 FirmClaw

在 src/index.ts 中注册你创建的工具：

typescript
// src/index.ts（摘录）

import { bashTool } from './tools/bash.js';
import { readTool } from './tools/read.js';
import { writeTool } from './tools/write.js';
import { editTool } from './tools/edit.js';
import { companyApiTool } from './tools/company-api.js';   // ← 导入你的工具
import { erpTool } from './tools/erp-tool.js';             // ←
import { dbQueryTool } from './tools/db-query.js';           // ←

async function main() {
  // ... LLM 初始化 ...

  const tools = new ToolRegistry();

  // 注册内置工具
  tools.register(bashTool);
  tools.register(readTool);
  tools.register(writeTool);
  tools.register(editTool);

  // 注册你的自定义工具
  tools.register(companyApiTool);    // ← 注册
  tools.register(erpTool);           // ← 注册
  tools.register(dbQueryTool);       // ← 注册

  // 设置权限策略
  const policy = new DefaultPermissionPolicy({ allowedPaths: [workDir] });
  tools.setPolicy(policy);

  // ... AgentLoop 初始化 ...
}
注意事项
  • 工具名称（name）必须全局唯一，不能与内置工具冲突
  • 注册顺序无影响，LLM 根据 description 自动选择
  • 需要额外 npm 包时，先 npm install <package>
  • 环境变量放在 .env 文件中（已在 .gitignore 中，不会泄露）

8. 安全与权限

内置权限策略

FirmClaw 的 DefaultPermissionPolicy 对文件工具和 bash 工具有内置限制：

工具类型保护措施
文件工具
(read/write/edit)		 路径白名单：只能在 allowedPaths 指定的目录内操作
系统目录保护：C:\Windows、C:\Program Files 默认禁止
敏感文件保护：不能写入 .env、credentials.json 等
bash 工具 命令黑名单：rm -rf /、format、shutdown 等
风险评估：rm/sudo → high，npm install → medium，查询 → low
自定义工具 默认放行——自定义工具不受内置策略限制
你需要在工具内部自行实现安全逻辑

自定义工具中的安全实践

typescript
// ✅ 好的安全实践

// 1. 输入校验
if (!/^SELECT\s/i.test(sql)) {
  return { content: 'Only SELECT allowed', isError: true };
}

// 2. 输出截断（避免消耗过多 context window）
if (resultStr.length > MAX_LEN) {
  return { content: resultStr.substring(0, MAX_LEN) + '...[truncated]' };
}

// 3. 超时保护
const controller = new AbortController();
setTimeout(() => controller.abort(), 30000);

function maskSensitive(data: Record<string, unknown>) {

    Object.entries(data).map(([k, v]) => [
      SENSITIVE_KEYS.some(s => k.toLowerCase().includes(s))
        ? [k, '***REDACTED***']
        : [k, v],
    ])
  );

9. 高级：钩子与审计

使用 HookManager 可以为工具添加审计日志、参数转换等能力：

typescript
import { HookManager } from './tools/hook-manager.js';

const hooks = new HookManager();

// Before Hook：为所有公司 API 调用添加追踪 header
hooks.registerBefore('company_api', (ctx) => {
  // 注入 trace_id
  return { args: { ...ctx.args, trace_id: crypto.randomUUID() } };
});

// Before Hook：禁止在非工作时间调用 ERP
hooks.registerBefore('erp_query', () => {
  const hour = new Date().getHours();
  if (hour < 9 || hour > 18) {
    return { deny: true, reason: 'ERP 只允许在工作时间 (9:00-18:00) 调用' };
  }
});

// After Hook：记录所有工具调用到审计日志
hooks.registerAfter('*', (ctx) => {
  auditLog.info({
    tool: ctx.toolName,
    args: ctx.args,
    riskLevel: ctx.riskLevel,
    timestamp: new Date().toISOString(),
  });
});

// 注册到 ToolRegistry
tools.setHookManager(hooks);

10. 最佳实践

实践说明
description 要详细 LLM 的工具选择完全依赖 description。写得越详细，LLM 调用越准确。包含：功能说明、参数含义、使用场景、返回格式。
截断大输出 API 返回可能很大，必须截断到合理长度（建议 10KB 以内），否则会消耗 context window 导致后续对话质量下降。
善用 enum 参数类型使用 enum 限制可选值，避免 LLM 传入无效参数（如 enum: ['GET', 'POST']）。
错误信息清晰 isError=true 时，提供有意义的错误描述，帮助 LLM 调整策略。
环境变量管理密钥 API Key、数据库密码等必须放在 .env 中，不要硬编码。.env 已在 .gitignore 中。
一个工具一个职责 不要把多个系统塞进一个工具。每个系统/服务应该有独立的工具，LLM 能更精准地选择。
幂等性 GET/查询操作应设计为幂等的，LLM 可能会重试失败的调用。
超时保护 所有外部调用必须设置超时（建议 10-30 秒），防止 LLM 循环卡死。

11. 常见问题

Q: LLM 不调用我创建的工具？

检查 description 是否清晰描述了工具的功能和使用场景。LLM 只有在认为需要时才会调用。可以在 SOUL.md 中添加提示，如"优先使用 company_api 工具查询订单数据"。

Q: 工具调用报 "Parameter validation failed"？

检查 parameters 中的 required 列表是否正确，以及 LLM 传入的参数类型是否与 schema 定义匹配。

Q: 自定义工具需要权限检查吗？

默认情况下，自定义工具不受 DefaultPermissionPolicy 限制（只对文件工具和 bash 工具生效）。如果你需要权限控制，请在 execute() 内部实现，或通过 Before Hook 实现。

Q: 如何在 Web UI / Gateway 中使用自定义工具？

Gateway 启动时会使用与 CLI 相同的 ToolRegistry 实例，所以自定义工具在 Web UI 中自动可用。

Q: 工具调用太慢怎么办？

1) 确保外部服务有索引；2) 在参数中增加过滤条件让 LLM 精准查询；3) 设置 AbortController 超时；4) 考虑缓存常用查询结果。

Q: 如何让 LLM 知道有哪些工具可用？

工具的 name、description、parameters 会被自动转换为 OpenAI function calling 格式，包含在每次 LLM 调用的 system prompt 中。你不需要手动维护工具列表。
