单位软件支撑展望
FirmClaw 如何赋能单位现有软件体系——场景、方案与实施路径

核心价值主张：FirmClaw 不是一个通用的聊天机器人，而是一个可以执行真实操作的 AI Agent。它能够读取文件、执行命令、修改代码、调用 API——这意味着它可以深度集成到单位现有的软件体系中。

二、场景一：内部文档智能检索

当前问题：单位的制度文件、技术文档、操作手册分散在共享盘、Wiki、OA 系统中，查找特定信息需要跨多个系统搜索，效率低下。

实现方案

架构设计

1. 文档同步

编写脚本，定期从各系统拉取文档到本地目录（PDF → 文本、Word → 文本）。FirmClaw 的 bash 工具可以执行 Python 脚本来完成。

2. 索引构建

利用 FirmClaw 内置的 BM25 搜索引擎，对所有文档建立全文索引。每次文档更新时增量更新索引。

3. 智能问答

用户提问时，Agent 自动检索相关文档片段，结合 LLM 的理解能力生成精准回答，并引用文档来源。

优势：所有数据在本地处理，不经过外部 API。文档内容不会泄露到互联网。

三、场景二：自动化运维巡检

当前问题：运维人员需要每天手动检查服务器状态、数据库连接、日志错误等，耗时且容易遗漏。

实现方案

基于 Heartbeat 的定时巡检

FirmClaw 已内置 Heartbeat 自主循环功能（Phase 5 实现），可以定时执行预设任务。

配置巡检任务

在 SOUL.md 中定义巡检规则：检查哪些服务、检查频率、异常阈值

Agent 自动执行

Heartbeat 触发后，Agent 自动执行 bash 命令检查服务状态（类似 systemctl status xxx）

智能分析

Agent 分析命令输出，判断是否异常。如果发现异常，通过通知渠道（邮件/消息）报告

生成巡检报告

每次巡检自动生成 Markdown 格式的巡检报告，存档备查

四、场景三：知识库问答助手

当前问题：新入职员工不了解单位的业务流程、技术规范、常见问题解答，需要反复询问同事。

实现方案

MEMORY.md 驱动的知识管理

FirmClaw 的 MEMORY.md 是一个结构化的长期记忆文件，天然适合做知识库。

MEMORY.md 示例

## 业务流程
[B001] 请假流程：OA提交→主管审批→HR备案
[B002] 报销流程：填写单据→部门审批→财务审核

## 技术规范
[T001] 内网系统必须使用 VPN 接入
[T002] 数据库操作必须走审计日志

## 常见问题
[Q001] VPN 连不上：检查证书是否过期

Agent 回答示例

用户：报销流程是什么？

Agent：根据知识库记录 [B002]，报销流程为：
1. 填写报销单据
2. 部门负责人审批
3. 财务部审核

如有其他疑问，可以继续问我。

五、场景四：代码审查助手

当前问题：Code Review 依赖资深开发者的时间，可能导致代码合并延迟。

实现方案

1. Agent 读取 Git Diff

通过 bash 工具执行 git diff main...feature，获取待审查的代码变更

2. 逐文件分析

Agent 读取变更涉及的源文件，理解上下文和业务逻辑

3. 多维度审查

检查代码规范、安全漏洞、性能问题、逻辑错误，生成结构化的审查报告

4. 自动提交评论

（未来通过 MCP 协议集成）将审查结果自动提交为 GitLab/GitHub PR 评论

六、实施路径规划

基于 FirmClaw 的现有能力和单位需求，建议按以下优先级分步实施：

Phase A 基础能力验证（1-2 周）

部署 FirmClaw，接入单位文档系统，验证文档检索和知识库问答的基本功能。选择一个小组进行试点。

Phase B 运维场景落地（2-4 周）

配置 Heartbeat 定时巡检任务，对接服务器监控脚本。建立异常通知机制。

Phase C 开发场景接入（1-2 月）

接入代码审查流程，与 GitLab/GitHub 集成。建立 FirmClaw 的 Web UI，方便团队成员使用。

Phase D 深度集成（2-3 月）

通过 MCP 协议对接 OA、邮件、IM 等系统。实现端到端的流程自动化。建立统一的 AI 工作台。

七、安全与合规考量

数据安全

FirmClaw 采用本地优先架构。所有对话历史、文档索引、记忆文件都存储在本地。可以搭配本地部署的 LLM（如 Ollama + Qwen），实现完全离线运行，数据不出内网。

操作安全

内置三级权限控制 + 人工审批机制。高危操作（删除文件、执行系统命令）必须经过人工确认。完整的审计日志记录每一次操作，可追溯可审查。

模型安全

Prompt Injection 防护机制防止通过工具返回结果进行注入攻击。权限白名单限制 Agent 只能访问授权的目录和服务。

合规建议

建议：① 对接本地 LLM 确保数据不出域；② 审计日志接入单位安全平台；③ 定期审查 MEMORY.md 中的敏感信息；④ 为不同用户设置不同的访问权限。
